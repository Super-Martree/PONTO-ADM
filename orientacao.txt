  alterações aparecem no ambiente de desenvolvimento:

  http://192.168.18.75:3004

  Quando quiser mandar as alterações para o original, rode:

  cd "C:\Users\user\Desktop\TOTTI\DEVELOPER\PONTO CPD ORIGINAL"
  npm.cmd run service:prepare
  net stop MartreePonto
  net start MartreePonto

  Resumo: você desenvolve/testa no 3004; quando aprovar, gera o build e reinicia o serviço para publicar no
  3002.


› pq nao é run build?


• Porque eu criei service:prepare como um nome mais explícito para “preparar o site que roda no serviço”.

  Mas hoje ele faz exatamente isso:

  "service:prepare": "npm.cmd run build --prefix frontend"

  Ou seja, estes dois comandos têm o mesmo efeito no seu projeto:

  npm.cmd run service:prepare

  e

  npm.cmd run build

  No package.json principal, o build também aponta para o build do frontend:

  "build": "npm.cmd run build --prefix frontend"

  Então pode usar assim sem problema:

  npm.cmd run build
  net stop MartreePonto
  net start MartreePonto

  Eu só usei service:prepare para ficar claro que é o passo antes de atualizar o serviço do Windows.
